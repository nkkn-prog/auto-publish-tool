# 禁止表現パターン

## 機密情報

以下のプレフィックス・パターンが含まれていないか確認する:

- `sk-` — Anthropic APIキー
- `secret_` — Notion APIトークン
- `ghp_` — GitHub Personal Access Token
- `gho_` — GitHub OAuth Token
- `xoxb-`, `xoxp-` — Slack トークン
- `Bearer ` + 英数字列 — Authorization ヘッダー
- `password`, `passwd`, `pwd` に続く `=` や `:` — パスワード露出

## 内部URL・内部情報

- `*.internal` — 内部ドメイン
- `*.local` — ローカルドメイン
- `192.168.*`, `10.*`, `172.16.*`〜`172.31.*` — プライベートIPアドレス
- `localhost:*` — ローカル開発環境URL

## 個人情報

- メールアドレスパターン: `xxx@xxx.xxx`
- 電話番号パターン: `0X0-XXXX-XXXX`, `+81-XX-XXXX-XXXX`
- 具体的な住所（番地を含む表現）

## 過度な断定表現

技術記事として不正確になりうる表現:

- 「絶対に」
- 「必ず」
- 「100%」
- 「間違いなく」
- 「確実に」
- 「完璧に」

これらが技術的事実として正しい文脈（例: 「この関数は必ず値を返す」等のコード仕様説明）であれば許容する。

## 更新方法

新しい禁止パターンを追加する場合は、該当セクションに追記してください。
