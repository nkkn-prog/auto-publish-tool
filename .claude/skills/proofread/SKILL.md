Claude APIが生成した記事ドラフト・X投稿文に対して、機械的校正チェックを実行してください。

## チェック対象の取得

$ARGUMENTS に対象が指定されていればそれを使い、指定がなければ `src/ai/prompts/` のプロンプトテンプレートを参照し、Claude APIの出力形式（JSON: `title`, `body`, `xPost1`〜`xPost3`）に基づいてチェックしてください。

## チェック項目

以下の各チェックを実行する。詳細ルールは補足ファイルを参照:

1. **表記ゆれ** — `terminology.md` の用語リストに基づき、正式表記に統一されているか確認
2. **Markdown構文** — コードブロック閉じ忘れ、リンク/画像構文不正、見出しレベル飛び、リスト記法不統一、空行不足
3. **文字数** — 記事本文: 2,000〜4,000文字目安、X投稿文: 各140文字以内。実際にカウントして報告
4. **禁止表現** — `prohibited-patterns.md` のパターンに基づき、機密情報・社内用語・個人情報・過度な断定が含まれていないか確認
5. **X投稿文** — 140文字制限、ハッシュタグ1〜2個、3パターンの差別化、型（問いかけ型・数字型・ストーリー型）の合致

## 出力形式

```
## 校正結果

### 表記ゆれ
- [箇所]: `誤表記` → `正式表記`
（なければ「問題なし」）

### Markdown構文
- [行/箇所]: 問題の説明
（なければ「問題なし」）

### 文字数
- 記事本文: X文字（目安: 2,000〜4,000文字）→ OK / NG
- xPost1: X文字 → OK / NG
- xPost2: X文字 → OK / NG
- xPost3: X文字 → OK / NG

### 禁止表現
- [箇所]: 検出された禁止表現
（なければ「問題なし」）

### X投稿文
- ハッシュタグ: OK / NG（詳細）
- パターン差別化: OK / NG（詳細）
- 型の合致: OK / NG（詳細）

### 総合判定
✅ 問題なし / ⚠️ 要修正（N件）
```

修正が必要な場合は、修正案も併せて提示してください。
